01Estructura del proyecto

Antes de tocar cualquier herramienta, tienes que saber dónde vive cada cosa. El proyecto tiene una estructura fija — si mueves archivos de sitio, los enlaces se rompen.

CONFD/ ← raíz del proyecto ├── index.html ← página principal con las cards ├── mapa.html ← mapa de conexiones D3 ├── case-builder.html ← herramienta: crear casos nuevos ├── css/ │ └── style.css ← estilos globales compartidos ├── js/ │ └── main.js ← reloj, timeline, hamburger ├── casos/ │ ├── valiant-thor.html ← Caso #001 │ ├── dulce-base.html ← Caso #002 │ ├── mk-ultra.html ← Caso #003 │ └── paperclip.html ← Caso #004 └── tools/ ├── index.html ← hub de herramientas ├── index-manager.html ← editar cards del índice ├── css-editor.html ← colores y fuentes ├── map-editor.html ← nodos del mapa ├── case-editor.html ← editar casos existentes ├── nav-editor.html ← menú de navegación └── guia.html ← este archivo
Regla de rutas: Los archivos en casos/ usan ../css/style.css y ../js/main.js. Los archivos en la raíz usan css/style.css y js/main.js. Las herramientas en tools/ tienen sus propios estilos embebidos.

02Las herramientas — para qué sirve cada una

Todas las herramientas funcionan con doble clic en el navegador, sin servidor. Generan código HTML que luego copias y reemplazas en el archivo correspondiente.

Case Builder
Crear un expediente nuevo desde cero. Genera el archivo .html completo listo para descargar.
→ case-builder.html (raíz)
Index Manager
Editar las tarjetas del índice: stamps, nivel de peligro, texto, orden. También filtros y stats del hero.
→ tools/index-manager.html
CSS Editor
Cambiar colores, fuentes y variables del proyecto. Preview en tiempo real. Genera el style.css actualizado.
→ tools/css-editor.html
Map Editor
Añadir nodos y conexiones al mapa sin tocar código D3. Genera el bloque graphData para pegar en mapa.html.
→ tools/map-editor.html
Case Editor
Editar un caso ya publicado. Pega el HTML existente, modifica visualmente y descarga el archivo actualizado.
→ tools/case-editor.html
Nav Editor
Añadir o renombrar páginas en el menú de navegación. Genera el bloque header para cada tipo de archivo.
→ tools/nav-editor.html

03Añadir un caso nuevo — flujo completo

Este es el flujo exacto de principio a fin. Son 4 pasos y ninguno requiere editar código a mano.

1. Case Builder
→
2. Index Manager
→
3. Map Editor
→
4. Subir archivos
1
Crear el expediente — Case Builder
Abre case-builder.html. Rellena número, título, categorías, nivel de peligro y estado. Añade los bloques de contenido que quieras (narrativa, timeline, citas...). Pulsa Generar y luego Descargar .html. Guarda el archivo en la carpeta casos/ con el nombre que pusiste.
2
Añadir la card al índice — Index Manager
Abre tools/index-manager.html. Pulsa "+ Añadir nuevo caso" al final de la lista. Rellena los campos con los mismos datos del caso. Ajusta el stamp y el peligro. Actualiza también el contador del hero (el número de Casos activos). Pulsa Generar, copia el HTML y reemplaza el contenido completo de index.html con él.
3
Añadir al mapa — Map Editor
Abre tools/map-editor.html. En el formulario de nodos, añade el caso principal (tipo: Caso, estado: Activo, URL: casos/tu-archivo.html). Añade también las personas y organizaciones clave. Conecta los nodos con el formulario de conexiones. Pulsa Generar graphData, copia el bloque y en mapa.html reemplaza todo el objeto const graphData = { ... }; con el nuevo.
4
Subir a GitHub Pages
Si tienes el proyecto en GitHub Pages: arrastra los archivos modificados al repositorio en GitHub.com o haz commit desde la terminal con git add . && git commit -m "Caso #00X" && git push. Los cambios aparecen en el sitio en 1-2 minutos.
Truco: el Index Manager tiene un campo "Nombre archivo HTML" — si lo rellenas con el nombre exacto del archivo (sin .html), el snippet para el index ya incluye la URL correcta del enlace.

05Actualizar el índice

El Index Manager reconstruye el index.html completo cada vez. Tiene los casos actuales cargados por defecto — si los cambias, recuerda que estás editando la copia local, no el archivo real.

FLUJOPasos para actualizar

1
Abre Index Manager
Los casos existentes ya están cargados. Si has añadido uno nuevo, pulsa "+ Añadir nuevo caso" al final y rellena sus datos.
2
Cambia lo que necesites
Puedes cambiar el stamp de cualquier caso (Clasificado → Abierto), el nivel de peligro, el texto del extracto, las categorías o el orden arrastrando con ↑↓. También puedes editar los filtros y los tres números del hero.
3
Genera y copia
Pulsa ▶ Generar. Revisa el preview. Pulsa ⎘ Copiar HTML.
4
Reemplaza el archivo
Abre index.html con un editor de texto (VS Code, Notepad++, cualquiera). Selecciona todo (Ctrl+A), pega (Ctrl+V), guarda. El archivo queda completamente reemplazado.
Importante: el Index Manager tiene los casos precargados con los datos del momento en que se creó la herramienta. Si has editado los casos manualmente desde entonces, los cambios en esos campos no se reflejan automáticamente — tendrás que actualizarlos a mano en el formulario antes de generar.

06Actualizar el mapa de conexiones

El mapa usa D3.js con un objeto graphData que contiene nodos y enlaces. El Map Editor genera ese bloque completo. Solo tienes que copiar y pegar en el lugar correcto de mapa.html.

DÓNDE PEGARLocalizar el bloque en mapa.html

Abre mapa.html con un editor de texto y busca (Ctrl+F) este texto exacto:

const graphData = {

Selecciona desde esa línea hasta el }; que cierra el objeto (unas 50-80 líneas más abajo según cuántos nodos tengas). Reemplaza todo ese bloque con el que genera el Map Editor.

URLCuándo poner URL y cuándo no

Solo los nodos de tipo case que ya tienen expediente publicado necesitan URL. El resto deja el campo vacío. La URL es relativa a la raíz del proyecto:

✓ Correcto: casos/mk-ultra.html ✗ Incorrecto: /casos/mk-ultra.html (la barra inicial lo rompe) ✗ Incorrecto: ../casos/mk-ultra.html (mapa.html está en raíz)

07Editar un caso ya publicado

Para cambios pequeños (corregir un texto, añadir un párrafo) usa el Case Editor. Para cambios grandes (reestructurar secciones completas) puede ser más rápido editar el HTML directamente.

CASE EDITORFlujo con la herramienta

1
Carga el archivo
Abre el archivo .html del caso con un editor de texto, selecciona todo, copia. En el Case Editor, pega en el campo de texto grande y pulsa ⬆ Cargar y extraer datos.
2
Revisa lo extraído
El extractor es automático pero no perfecto. Revisa que el título, subtítulo y bloques se hayan cargado correctamente. Corrige lo que no esté bien antes de continuar.
3
Edita y genera
Modifica lo que necesites. Pulsa ▶ Generar, revisa el preview, y descarga o copia el HTML resultante.

08Cambiar la navegación

Cuando quieras activar las páginas "Archivo" y "Acerca de" que ahora son #, o añadir una sección nueva al menú, usa el Nav Editor.

1
Edita los enlaces
En el Nav Editor, cambia el texto o la URL de cualquier enlace. Reordena con ↑↓. Añade nuevos con el botón "+".
2
Selecciona el tipo de página
Elige entre Raíz (para index.html y mapa.html), Caso (para archivos en /casos/) o Tool (para herramientas). Las rutas relativas cambian según esto.
3
Genera y reemplaza
Genera, copia el bloque. En cada archivo HTML que quieras actualizar, busca <header class="site-header"> y reemplaza todo hasta </header> con el nuevo bloque.
Recuerda: si cambias la navegación, tienes que aplicar el cambio en todos los archivos del proyecto. En la raíz (index.html, mapa.html) y en cada archivo de casos/. Los casos necesitan el bloque de tipo "Caso", no "Raíz".

09Cambiar diseño y colores

El CSS Editor genera el style.css completo. Cuando quieras cambiar el color ámbar por otro, probar fuentes diferentes o ajustar los fondos, es el camino.

1
Ajusta en el panel izquierdo
Los pickers de color actualizan el preview en tiempo real. Prueba combinaciones hasta que te guste el resultado. El ámbar es el color dominante — cambiar --amber, --amber-bright y --amber-dim juntos cambia toda la personalidad visual del proyecto.
2
Genera el CSS completo
Pulsa ▶ Generar CSS completo, luego cambia a la pestaña HTML y copia todo. Abre css/style.css, selecciona todo, pega, guarda.
Precaución: el CSS Editor reemplaza el archivo completo. Si has hecho cambios manuales en style.css que no están en el editor, se perderán. Haz una copia de seguridad antes.

11Stamps y estados

ClaseVisualCuándo usarlo
stamp-openVerde — "Abierto"El expediente está publicado y accesible
stamp-classifiedRojo — "Clasificado"El expediente existe pero está "bloqueado" narrativamente
stamp-pendingÁmbar — "Pendiente"En preparación, no hay expediente aún
Regla práctica: si el archivo .html del caso ya existe en /casos/ → usa stamp-open. Si el caso aparece en el index como "próximamente" sin archivo → usa stamp-classified o stamp-pending y añade la clase coming-soon al div de la card.

12Errores más comunes

  • El caso sale en el index pero el enlace no funciona — el nombre del archivo en el href no coincide con el nombre real del .html. Son case-sensitive: MK-Ultra.html ≠ mk-ultra.html.
  • Los estilos del caso no se cargan (página sin diseño) — el archivo está en /casos/ pero la ruta al CSS dice css/style.css en vez de ../css/style.css. Cada caso necesita el ../ para subir un nivel.
  • El mapa no muestra los nodos nuevos — pegaste el graphData en el lugar equivocado o con sintaxis rota. Abre la consola del navegador (F12 → Console) y busca el error. Suele ser una coma sobrante o que falta el }; de cierre.
  • El Index Manager genera un index que no tiene todos los casos — los casos están precargados en la herramienta con los datos del momento en que se creó. Si has añadido casos nuevos a mano, tienes que añadirlos también en el Index Manager antes de generar.
  • La timeline no se expande al hacer clic — falta el script <script src="../js/main.js"></script> al final del body, o la ruta es incorrecta.
  • Las imágenes del caso no cargan — las URLs de imágenes externas a veces cambian o expiran. Cada caso tiene un onerror que muestra "IMAGEN CLASIFICADA" si la imagen falla. Para imágenes permanentes, usa Wikimedia Commons con la URL completa de commons.wikimedia.org.
  • El menú hamburger en móvil no se abre — falta el botón <button class="nav-toggle"> antes del <div class="nav-status"> en el header del archivo. Cada caso necesita tener ese botón.

13Cuándo necesitas a Claude — y cuándo no

La idea de toda esta infraestructura es que puedas trabajar el proyecto solo. Aquí tienes claro qué puedes hacer sin ayuda y qué no.

SIN CLAUDELo que puedes hacer completamente solo

  • Crear un caso nuevo — Case Builder completo, sin excepción
  • Cambiar el stamp de un caso (Clasificado → Abierto) — Index Manager
  • Añadir o reordenar filtros en el índice — Index Manager
  • Actualizar el contador de casos activos — Index Manager, campo "Número" en el hero stat
  • Añadir nodos y conexiones al mapa — Map Editor
  • Corregir texto en un caso ya publicado — Case Editor o edición directa del HTML
  • Cambiar colores y fuentes del proyecto — CSS Editor
  • Activar páginas del menú ("Archivo", "Acerca de") — Nav Editor
  • Añadir un bloque redactado o una cita a un caso — editar el HTML directamente con la referencia de arriba

CON CLAUDECuándo tiene sentido pedirme ayuda

  • Escribir el contenido narrativo de un caso que no conoces bien — investigación y redacción
  • Añadir una funcionalidad nueva que no existe en ninguna herramienta (un nuevo tipo de bloque, una página entera nueva)
  • Depurar un error raro que no aparece en la lista de errores comunes
  • Rediseñar una sección completa del proyecto (nuevo layout, nueva página)
  • Integrar algo externo — API, base de datos, sistema de comentarios
Para conversaciones con Claude: abre siempre una conversación nueva por tema. "Necesito el contenido del caso Majestic 12" es una conversación. "El Map Editor tiene un bug en las conexiones" es otra. Nunca mezcles en el mismo hilo contenido creativo con correcciones de código — cada tema por separado consume mucho menos contexto.
